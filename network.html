Network Architecture

MonadBFT Consensus

MonadBFT is a pipelined Byzantine Fault Tolerant consensus protocol designed for high throughput and single-slot finality. Unlike Ethereum's multi-slot finality mechanism, MonadBFT achieves finality within one block (~1 second), making Monad suitable for applications that require rapid settlement guarantees such as exchanges, payment systems, and latency-sensitive dApps. The protocol is resilient to up to one-third of validators being Byzantine (faulty or malicious).

Parallel Execution Engine

The Monad execution engine runs EVM transactions in parallel using optimistic concurrency control. Transactions are speculatively executed across multiple CPU cores simultaneously. If a conflict is detected (two transactions touching the same storage slot in an incompatible way), the conflicting transaction is re-executed sequentially to ensure correctness. In typical DeFi workloads the vast majority of transactions are conflict-free, yielding near-linear throughput scaling with available CPU cores.

Local Mempool Design

Monad does not maintain a global, shared mempool. Each node maintains its own local pending transaction pool. This design reduces propagation overhead and improves throughput, but it means that a transaction submitted via one RPC endpoint may not be visible in the mempool of a different endpoint until it is included in a block. Applications should not depend on cross-node mempool visibility and should instead track transactions by hash after submission.

Connection Configuration

Configure an ethers.js provider for Monad mainnet. Always use EIP-1559 gas pricing. 

const provider = new ethers.JsonRpcProvider(
  "https://monad-mainnet.dwellir.com/YOUR_API_KEY",
  { chainId: 143, name: "monad" }
);

// Always use EIP-1559 gas pricing on Monad
const feeData = await provider.getFeeData();
const tx = await wallet.sendTransaction({
  to: recipientAddress,
  value: ethers.parseEther("1.0"),
  maxFeePerGas: feeData.maxFeePerGas,
  maxPriorityFeePerGas: feeData.maxPriorityFeePerGas,
});
await tx.wait();
