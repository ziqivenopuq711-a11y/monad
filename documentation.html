Monad Documentation

Welcome to the official Monad developer documentation. Monad is a high-performance EVM Layer 1 blockchain delivering 10,000+ TPS, 1-second block times, and full Ethereum compatibility. This documentation covers everything you need to build, deploy, and scale applications on Monad.

Getting Started

Monad is fully EVM-compatible, which means that if you have experience building on Ethereum, you can deploy to Monad with zero code changes. Configure your network settings and you are ready.

  1. 1. Configure your wallet or development environment with Monad network settings (Chain ID 143, RPC endpoint from Dwellir).
  2. 2. Obtain MON tokens (the native gas token) via the official bridge or testnet faucet for development.
  3. 3. Deploy your contracts using Hardhat, Foundry, or Remix — all standard Ethereum tools work on Monad.
  4. 4. Connect your frontend with ethers.js, viem, or web3.js using a Dwellir RPC endpoint.

Key Concepts

Parallel Execution

Unlike traditional EVM chains that execute transactions sequentially, Monad uses optimistic parallel execution. Transactions are executed concurrently across multiple threads, with conflict detection ensuring correctness. This enables the dramatic throughput increase to 10,000+ TPS while maintaining full EVM compatibility and deterministic state transitions.

Local Mempool Design

Monad does not have a global mempool. Each RPC node maintains its own local mempool, which means a pending transaction submitted to one node may not be visible via another node until it is included in a block. Applications should track transactions by hash and poll for confirmation rather than relying on cross-node mempool visibility.

MonadBFT Consensus

MonadBFT is Monad's high-performance Byzantine Fault Tolerant consensus mechanism. It is a pipelined BFT protocol that achieves single-slot finality, enabling the 1-second block time that makes Monad suitable for latency-sensitive applications such as high-frequency trading and gaming where rapid settlement is critical.

EIP-1559 Gas Pricing

Monad uses the EIP-1559 transaction fee mechanism with a base fee that adjusts dynamically based on network demand. Always use maxFeePerGas and maxPriorityFeePerGas fields for optimal gas pricing. Legacy transactions are supported but EIP-1559 is strongly recommended.
