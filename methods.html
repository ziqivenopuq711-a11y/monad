JSON-RPC Methods Reference

Monad supports the standard Ethereum JSON-RPC API plus Monad-specific extensions for mempool inspection. All methods are available via HTTPS and WebSocket transports on both mainnet and testnet.

Standard Ethereum Methods

All standard Ethereum JSON-RPC methods are supported. The following table lists the most commonly used methods when building on Monad.

Method Description
eth_blockNumber Returns the current block number. Use to track chain head and confirm transaction inclusion.
eth_getBalance Returns the MON balance of an address at a specified block. Supports "latest", "earliest", and numeric block tags.
eth_call Executes a read-only message call without creating a transaction. Used for contract view and pure functions.
eth_sendRawTransaction Submits a signed, RLP-encoded transaction to the network. Returns the transaction hash immediately upon receipt.
eth_getTransactionReceipt Returns the receipt of a mined transaction including status (0 or 1), gas used, and event logs emitted.
eth_getTransactionByHash Returns transaction details by hash: sender, receiver, value, input data, nonce, and EIP-1559 gas fields.
eth_getBlockByNumber Returns full block data by block number. Set the second parameter to true to include full transaction objects.
eth_getLogs Returns logs matching a filter: address, topics, and block range. Essential for event-driven indexing.
eth_estimateGas Estimates the gas required for a transaction. Add a 20% buffer before submitting to avoid out-of-gas errors.
eth_gasPrice Returns the current gas price. Use maxFeePerGas / maxPriorityFeePerGas for EIP-1559 transactions instead.
eth_chainId Returns the chain ID as a hex string. Returns 0x8f (decimal 143) for Monad mainnet.
eth_getCode Returns the deployed bytecode of a contract at a given address and block number.
eth_getStorageAt Returns the value at a specific storage slot of a contract. Useful for low-level state inspection.
eth_getTransactionCount Returns the nonce of an address. Use "pending" tag to include transactions in the local mempool.
net_version Returns the network ID as a decimal string. Returns "143" for Monad mainnet and "10143" for testnet.

Monad-Specific Methods

Due to Monad's local mempool design, standard txpool methods are scoped to the local node. Monad exposes additional methods for querying the local pending transaction pool by address or hash. These methods are useful for applications that need to inspect the pending state of a specific address or track the status of a submitted transaction before it is included in a block.

Method Description
txpool_statusByAddress Returns the count of pending and queued transactions in the local mempool for a specific sender address.
txpool_statusByHash Returns the status of a specific transaction in the local mempool: pending, queued, or not found (already mined or not submitted to this node).

Request Examples

eth_blockNumber — cURL

curl -X POST https://monad-mainnet.dwellir.com/YOUR_API_KEY \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}'

# Response
# {"jsonrpc":"2.0","id":1,"result":"0x1a4b2c"}

eth_getBalance — JavaScript

const response = await fetch(
  "https://monad-mainnet.dwellir.com/YOUR_API_KEY",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      jsonrpc: "2.0",
      method: "eth_getBalance",
      params: ["0xYourAddress", "latest"],
      id: 1,
    }),
  }
);
const { result } = await response.json();
const balanceMON = Number(BigInt(result)) / 1e18;
console.log(`Balance: ${balanceMON} MON`);

txpool_statusByAddress — cURL

curl -X POST https://monad-mainnet.dwellir.com/YOUR_API_KEY \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "txpool_statusByAddress",
    "params": ["0xYourAddress"],
    "id": 1
  }'

# Response
# {"jsonrpc":"2.0","id":1,"result":{"pending":"0x2","queued":"0x0"}}

eth_getLogs — JavaScript

// Fetch Transfer events for an ERC-20 contract
const logs = await provider.getLogs({
  address: "0xTokenContractAddress",
  topics: [
    ethers.id("Transfer(address,address,uint256)"),
  ],
  fromBlock: "latest",
  toBlock: "latest",
});
console.log(`Found ${logs.length} Transfer events`);
