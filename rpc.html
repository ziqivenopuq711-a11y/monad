Authentication

Dwellir RPC endpoints use API key authentication embedded in the URL path. This approach is compatible with all standard Ethereum libraries without additional configuration — simply replace YOUR_API_KEY in the URL.

  1. 1. Sign up or log in at dashboard.dwellir.com
  2. 2. Create an application and select Monad as the network (mainnet or testnet)
  3. 3. Copy your API key and replace YOUR_API_KEY in the endpoint URL
Security: Treat your API key like a password. Do not commit it to source control. Use environment variables (process.env.DWELLIR_API_KEY) in your application code.

Best Practices

Batch Requests

Combine multiple RPC calls into a single HTTP request to reduce latency and improve throughput for bulk data fetching operations.

// ethers.js v6 — parallel calls
const [balance, block] = await Promise.all([
  provider.getBalance(address),
  provider.getBlockNumber(),
]);

Connection Pooling

Reuse a single provider instance across your application instead of creating a new connection per request to minimize overhead.

// Create once, export and reuse
const provider = new ethers.JsonRpcProvider(
  process.env.DWELLIR_API_KEY
);
export default provider;

Caching

Cache responses for immutable data such as block by hash and transaction receipts to avoid redundant RPC round-trips.

// Receipts are immutable once confirmed
const receipt = cache.get(txHash)
  ?? await provider.getTransactionReceipt(txHash);
cache.set(txHash, receipt);

Retry with Backoff

Implement exponential backoff for transient failures to build resilient production applications that handle network hiccups gracefully.

async function retry(fn, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try { return await fn(); }
    catch { await sleep(2 ** i * 100); }
  }
  throw new Error("Max retries exceeded");
}
