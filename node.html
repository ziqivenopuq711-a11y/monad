Running a Monad Node

Operating a Monad node allows you to participate in the network, serve RPC requests, or maintain an independent copy of blockchain history. This guide covers hardware requirements, node types, and setup.

Node Types

Full Node

A full node downloads and validates all blocks from genesis, maintains the current world state, and can serve JSON-RPC requests. It does not store the complete historical state of every block, only the latest state and enough history to support reorgs and recent queries. Full nodes are suitable for most dApp backends and RPC serving use cases.

Recommended for
RPC serving, validation, dApp backends

Archive Node

An archive node stores the complete historical state at every block height. This enables queries like eth_getBalance at any historical block number. Archive nodes require significantly more storage and compute than full nodes but are necessary for analytics platforms, blockchain indexers, and historical research.

Recommended for
Analytics, indexers, historical queries

Hardware Requirements

Full Node
CPU16+ cores (x86-64)
RAM64 GB minimum
Storage2 TB NVMe SSD
Network1 Gbps dedicated
Archive Node
CPU32+ cores
RAM128 GB minimum
Storage10+ TB NVMe SSD
Network10 Gbps dedicated
Note: Monad's parallel execution engine is highly CPU and RAM intensive. Ensure your hardware meets the minimum requirements before attempting to run a production node.

Setup Overview

Refer to the official Monad documentation and GitHub releases for the latest node client binaries and configuration files. The general setup process follows these steps:

  1. 1
    Download the Monad client
    Obtain the latest release of the Monad node binary from the official GitHub releases page. Verify the checksum before running.
  2. 2
    Configure the node
    Edit the configuration file to set your data directory, network ID (143 for mainnet), RPC listen port, and peer discovery settings including bootnodes.
  3. 3
    Start syncing
    Run the client and allow it to sync from genesis or from a trusted chain snapshot. Initial sync may take several hours depending on hardware and bandwidth.
  4. 4
    Enable RPC
    Enable the HTTP-RPC and/or WebSocket-RPC interface in the configuration file to expose JSON-RPC endpoints for your dApps and tooling.
  5. 5
    Monitor
    Use Prometheus and Grafana with the built-in metrics endpoint to monitor sync status, peer count, memory usage, and block processing rate in real time.
